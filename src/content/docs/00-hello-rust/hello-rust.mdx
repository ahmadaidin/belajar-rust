Halaman ini menjelaskan cara memasang toolchain Rust dan membiasakan diri dengan workflow `cargo` sebelum masuk ke topik bahasa.

## Prasyarat

- Shell atau terminal (PowerShell, Bash, Zsh, Fish)
- Koneksi internet untuk mengunduh `rustup`

## Install Rust

Rust biasanya diinstall lewat `rustup` — installer dan version manager sekaligus:

```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

Ikuti instruksi di terminal, lalu muat ulang environment:

```bash
source $HOME/.cargo/env
```

Cek versi yang terpasang:

```bash
rustc --version
cargo --version
rustup --version
```

Output yang wajar terlihat seperti:

```text
rustc 1.96.1
cargo 1.96.1
rustup 1.27.1
```

## Membuat Project Baru

`cargo` memudahkan pembuatan project. Perintah di bawah ini membuat folder `hello-rust` dengan struktur standar:

```bash
cargo new hello-rust
cd hello-rust
cargo run
```

Kalau berhasil, outputnya:

```text
   Compiling hello-rust v0.1.0
    Finished `dev` profile [unoptimized + debuginfo]
     Running `target/debug/hello-rust`
Hello, world!
```

## Struktur Project Cargo

Buka project yang baru dibuat dan perhatikan file-file ini:

| File / Folder | Fungsi |
|---------------|--------|
| `Cargo.toml` | Metadata project dan daftar dependencies |
| `src/main.rs` | Entry point untuk binary (`cargo run`) |
| `target/` | Hasil build; bisa diabaikan, biasanya masuk `.gitignore` |
| `Cargo.lock` | Snapshot dependency yang dipakai; muncul setelah build pertama |

Contoh `Cargo.toml` minimal:

```toml
[package]
name = "hello-rust"
version = "0.1.0"
edition = "2021"

[dependencies]
```

Field `edition` menentukan versi aturan bahasa Rust. Saat ini umumnya dipakai `2021`.

## 4 Perintah Utama

Biasakan menggunakan perintah-perintah ini dalam siklus development:

| Perintah | Fungsi |
|----------|--------|
| `cargo check` | Cek kode tanpa build (paling cepat) |
| `cargo build` | Compile project |
| `cargo run` | Compile + jalankan |
| `cargo test` | Jalankan test |

Ada juga perintah bantu yang sering dipakai:

| Perintah | Fungsi |
|----------|--------|
| `cargo fmt` | Format kode secara otomatis |
| `cargo clippy` | Linter bawaan Rust; memberi saran idiomatic |
| `cargo doc --open` | Bangkitkan dan buka dokumentasi lokal |
| `cargo clean` | Hapus folder `target/` untuk memulai build dari awal |

## Setup Editor

Rust paling nyaman ditulis dengan `rust-analyzer`.

### VS Code / Cursor

Install extension **rust-analyzer** (bukan yang bernama "Rust"). Setelah itu VS Code akan memberikan:

- Autocomplete
- Error inline
- Goto definition
- Refactoring rename

### Lainnya

`rust-analyzer` juga tersedia untuk Helix, Neovim, Zed, Emacs, dan Sublime Text. Intinya: install LSP server lewat `rustup`:

```bash
rustup component add rust-analyzer
```

Kemudian konfigurasikan editor untuk menjalankan binary tersebut.

## Membaca Error Compiler

Salah satu kekuatan Rust adalah pesan error yang jelas. Cobalah ubah `src/main.rs` jadi kode yang sengaja salah:

```rust
fn main() {
    let x = 5;
    x = 6;
    println!("{x}");
}
```

Lalu jalankan:

```bash
cargo check
```

Rust akan memberitahu bahwa `x` tidak bisa diassign ulang karena tidak mutable. Biasanya compiler langsung menyarankan menambahkan `mut`.

## Tips Workflow

1. **Mulai dari `cargo check`, bukan `cargo build`.** Jauh lebih cepat karena tidak memproduksi binary.
2. **Jalankan `cargo fmt` dan `cargo clippy` sebelum commit.** Ini membuat kode konsisten dan mengurangi bug sederhana.
3. **Baca error dari atas ke bawah.** Pesan pertama biasanya sudah cukup untuk memperbaiki masalah.
4. **Gunakan Rustlings untuk latihan.** Kumpulan latihan kecil yang langsung dikoreksi.
